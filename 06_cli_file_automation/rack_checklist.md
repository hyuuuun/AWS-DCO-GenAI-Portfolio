# [교육용 샘플 데이터 사용]

## 교육용 랙 확인 체크리스트 (SAMPLE_RACK_CHECKLIST)

본 체크리스트는 실제 물리 장비의 점검 절차가 아니며, 교육 과정 중 제공된 샘플 로그와 샘플 티켓을 읽고 기록할 항목들을 정리하기 위한 교육용 체크리스트입니다.

### 1. 기본 정보 확인 항목
- [ ] **발생 시간 확인**: 샘플 티켓 또는 로그에 나타난 이벤트 발생 시간 기록
  - 기록란: (예: `2026-07-07T15:00:00+09:00`)
- [ ] **샘플 장비명 확인**: 대상 샘플 장비명 기록 (반드시 `SAMPLE_` 또는 `EDU_` 접두어 사용 여부 확인)
  - 기록란: (예: `SAMPLE_TOR_SW_01`)
- [ ] **Ticket 상태 확인**: 현재 티켓의 상태(Open, In-Progress, Pending 등) 확인 및 기록
  - 기록란: (예: `EDU_STATUS_OPEN`)

### 2. 이벤트 패턴 및 반복 분석
- [ ] **반복 여부 확인**: 동일 장비 혹은 동일 인터페이스에서 유사한 CRC 에러나 Link Down 로그가 이전 샘플 데이터에서 관찰되었는지 확인
  - 기록란: (예: `동일 인터페이스에서 최근 24시간 내 3회 SAMPLE_RECURRENCE 관찰됨`)

### 3. 장애 원인 분석 및 주의사항
- [ ] **원인 단정 금지**: CRC 에러 및 Link Down 발생 시, 케이블 불량/포트 불량 등으로 성급히 단정 짓지 않고 논리적 가능성을 열어두었는지 체크
  - 주의: "케이블 고장으로 판명됨" 대신 "물리 계층 오류 가능성이 존재하므로 추가 진단 필요"와 같이 서술할 것.

### 4. 문서화 및 후속 조치
- [ ] **보고서 초안 작성 메모**: 문제의 현상, 일시, 샘플 장비명, 관찰된 로그를 포함한 요약본 작성 여부
  - 작성 메모란: 
    * *예시: SAMPLE_TOR_SW_01의 Ethernet1/1 포트에서 CRC 에러 증가 후 Link Down 발생. 외부 점검이나 실서버 접속 없이 제공된 샘플 로그 데이터만으로 분석을 진행함.*
