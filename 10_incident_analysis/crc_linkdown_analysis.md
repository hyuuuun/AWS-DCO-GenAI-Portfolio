# CRC Error·Link Down 교육용 Incident 분석

- 차시: 10차시
- 작성자: (본인 이름 입력)
- 사용 파일: `09_log_analysis_script/sample_dco_log.txt`
- 사용 자료: 교육용 샘플 데이터
- 분석 기준: 원본 로그에 직접 기록된 내용과 가능한 원인 추정을 구분함

## 1. 분석 대상

- 날짜: 2026-07-03
- 샘플 장비명: `SAMPLE_TOR_SW_01`
- 관리 IP: `192.0.2.1`
- 인터페이스: `Gi0/1`
- Ticket ID: `EDU-TKT-2026-0003`
- 분석 시간: 03:05~03:30
- 장애 영향: 로그에 `Connection to server lost`가 기록되어 서버 연결이 끊김

## 2. 사건 전 상태

- [사실] 03:00에 `SAMPLE_TOR_SW_01`의 상태가 정상(`Normal heartbeat`)으로 기록되었습니다.
- [미확인] 03:05 이전의 `Gi0/1` CRC 카운터 값이나 광 신호 값은 현재 로그에 없습니다.

| 시간 | 심각도 | 이벤트 | 확인된 내용 |
|---|---|---|---|
| 03:05 | WARNING | CRC error 증가 | [사실] `Gi0/1`의 CRC 오류 카운터가 5분 안에 154까지 증가함 |
| 03:06 | ERROR | Link Down | [사실] `Gi0/1` 상태가 DOWN으로 변경되고 서버 연결이 끊김 |

| 03:25 | INFO | Maintenance completed | [사실] LC-LC 멀티모드 패치 코드를 교체하고 포트 커넥터를 청소함 |
| 03:26 | INFO | Link Up | [사실] `Gi0/1`과 line protocol이 UP으로 변경됨 |
| 03:30 | INFO | Normal heartbeat | [사실] 시스템이 정상이며 `Gi0/1`의 CRC 오류가 0으로 기록됨 |

## 4. 로그에서 확인된 사실

- [사실] CRC 오류 증가는 `SAMPLE_TOR_SW_01`의 `Gi0/1`에서 발생했습니다.
- [사실] CRC 오류 카운터가 5분 안에 154까지 증가한 뒤 1분 후 Link Down이 기록되었습니다.
- [사실] Link Down 시점에 연결 대상 서버와의 연결 손실이 기록되었습니다.
- [사실] 현장 케이블 팀이 패치 코드를 교체하고 포트 커넥터를 청소했습니다.
- [사실] 작업 완료 1분 후 링크가 복구되었고, 4분 뒤 정상 상태와 CRC 오류 0이 기록되었습니다.
- [사실] 사건의 첫 이상 기록부터 정상 상태 확인까지 걸린 시간은 25분입니다.
- [사실] 이 파일은 교육용 샘플 로그이며 실제 운영 장비의 기록이 아닙니다.

### 9차시 분석 결과와 교차검증

- [사실] 전체 140줄의 심각도 집계는 `INFO` 135줄, `WARNING` 3줄, `ERROR` 2줄로 `incident_summary.md`와 일치합니다.
- [사실] 원본 로그에서 `SAMPLE_TOR_SW_01`은 총 48줄입니다.
- [사실] 03:05~03:30 사건 구간은 7줄이며 `WARNING` 1줄, `ERROR` 1줄, `INFO` 5줄입니다.
- [사실] 이 사건의 CRC 증가 이벤트와 Link Down 이벤트는 각각 1줄입니다.

## 5. 가능한 원인

현재 로그만으로 하나의 원인을 확정할 수 없습니다. 아래 항목은 모두 가능한 원인입니다.

### 가능성 1: 패치 코드 또는 커넥터 접촉 상태 문제

- 가능한 원인: [추정] LC-LC 멀티모드 패치 코드의 손상, 과도한 굴곡, 느슨한 체결 또는 커넥터 오염이 신호 품질을 낮췄을 수 있습니다.
- 로그에서 찾은 근거: [사실] CRC 증가와 Link Down 이후 패치 코드 교체 및 커넥터 청소가 수행되었고, 그 뒤 링크가 복구되었습니다.
- 판단: [추정] 조치와 복구 시점의 연관성 때문에 비교적 가능성이 높지만, 교체와 청소를 동시에 수행했으므로 어느 조치가 영향을 주었는지는 알 수 없습니다.
- 추가로 필요한 정보: 교체한 케이블의 외관·굴곡·단선 검사 결과, 커넥터 오염 상태, 재장착 전후 측정값, 교체 케이블 정보

### 가능성 2: 광 트랜시버 또는 광 신호 품질 문제

- 가능한 원인: [추정] 스위치 또는 상대 장비의 광 트랜시버 이상이나 송수신 광 파워 저하가 CRC 증가와 링크 단절에 영향을 주었을 수 있습니다.
- 로그에서 찾은 근거: [추정] CRC 오류와 Link Down은 물리 계층의 신호 품질 문제에서도 나타날 수 있습니다.
- 판단: [추정] 가능한 원인이지만, 로그에 트랜시버 경보나 광 파워 값이 없어 직접적인 근거는 부족합니다.
- 추가로 필요한 정보: 양쪽 트랜시버의 Tx/Rx 광 파워, 온도, 전압, 제조사 및 모델, 알람 기록, 교차 교체 테스트 결과

### 가능성 3: 스위치 포트 또는 상대 장비 포트 문제

- 가능한 원인: [추정] `Gi0/1` 자체 또는 연결된 서버·네트워크 장비의 포트/NIC 상태가 불안정했을 수 있습니다.
- 로그에서 찾은 근거: [사실] 이상 기록이 `Gi0/1`에 집중되어 있고 Link Down 메시지는 서버 연결 손실을 함께 기록합니다.
- 판단: [미확인] 현재 로그에는 반대편 장비의 상태와 포트 진단 결과가 없어 케이블 구간 문제와 구분할 수 없습니다.
- 추가로 필요한 정보: 양쪽 포트의 오류 카운터와 상태 변경 이력, 시스템 로그, NIC·포트 진단 결과, 다른 정상 포트로의 교차 연결 결과

## 6. 현재 로그만으로 확인할 수 없는 내용

- [미확인] CRC 증가가 Link Down의 직접적이고 유일한 원인인지 여부
- [미확인] 패치 코드 교체와 커넥터 청소 중 어떤 조치가 복구에 기여했는지 여부
- [미확인] 케이블, 커넥터, 트랜시버, 스위치 포트, 상대 장비 포트 가운데 실제 최초 원인이 무엇인지
- [미확인] 사건 전의 CRC 누적값과 같은 현상의 과거 반복 여부
- [미확인] 상대 장비의 이름, 포트, 당시 로그 및 물리 상태
- [미확인] 복구 후 03:30 이후 장시간 동안 CRC 오류가 재발하지 않았는지 여부
- [미확인] 장애로 영향받은 서버, 서비스, 사용자 범위와 실제 업무 영향

## 7. 추가로 확인하면 좋은 정보

1. 사건 전후 `Gi0/1`의 CRC, input error, frame error, interface reset 카운터 추이
2. 상대 장비 포트의 같은 시간대 오류 카운터와 Link Down/Up 로그
3. 양쪽 광 트랜시버의 Tx/Rx 광 파워와 경보 기록
4. 패치 코드 교체 전 케이블 및 커넥터 검사 결과와 사진 또는 측정값
5. 사건 이전의 동일 포트 장애 및 CRC 오류 이력
6. 복구 후 CRC 카운터 추이와 장시간 안정성 모니터링 결과
7. 장애 영향을 받은 서버·서비스와 연결 구성 정보
8. 티켓의 현장 작업 메모, 교체 부품 정보 및 최종 원인 판정

## 8. Ticket 전달 시 포함하면 좋은 내용

- 장비·포트·관리 IP: `SAMPLE_TOR_SW_01`, `Gi0/1`, `192.0.2.1`
- 티켓 번호와 시간: `EDU-TKT-2026-0003`, 2026-07-03 03:05~03:30
- 현상과 영향: 5분 안에 CRC 오류 카운터가 154까지 증가, Link Down, 서버 연결 손실
- 핵심 타임라인: CRC 증가 → Link Down → 티켓 생성·에스컬레이션 → 현장 조치 → Link Up → CRC 0
- 수행 조치: LC-LC 멀티모드 패치 코드 교체 및 포트 커넥터 청소
- 현재 상태: 03:26 Link Up, 03:30 정상 heartbeat 및 CRC 오류 0
- 요청할 확인 사항: 상대 포트 로그, 광 파워, 트랜시버·포트 상태, 재발 여부
- 주의 사항: 케이블 문제로 확정하지 말고 현재 증거와 미확인 항목을 함께 전달

## 12. AI 결과를 검토해 직접 수정한 내용

1. 원본 로그의 시간, 장비, 포트, 티켓 번호, CRC 수치와 작업 내용을 다시 대조했습니다.
2. 시간순 표에 심각도(`WARNING`, `ERROR`, `INFO`)와 장애 영향을 추가했습니다.
3. 패치 코드 문제를 확정 원인으로 표현하지 않고 케이블·트랜시버·포트의 세 가지 가능성으로 분리했습니다.
4. 조치 후 복구는 사실이지만 인과관계와 최초 원인은 미확인임을 명시했습니다.
5. 티켓 전달 시 필요한 현재 상태, 요청 사항, 미확인 내용을 추가했습니다.

## 13. 분석 결론

[사실] CRC 오류 증가, Link Down, 현장 케이블 관련 조치, Link Up과 CRC 오류 0의 순서는 원본 로그에서 확인됩니다. [추정] 패치 코드나 커넥터를 포함한 물리 연결 구간이 사건과 관련되었을 가능성이 있습니다. [미확인] 그러나 여러 조치가 함께 수행되었고 광 신호·트랜시버·양쪽 포트 진단 자료가 없으므로 최초 원인을 하나로 확정할 수 없습니다.
